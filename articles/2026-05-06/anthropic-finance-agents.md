## 核心发布内容

Anthropic 于 2026 年 5 月 5 日正式宣布推出十个现成可用的金融服务业 Agent 模板，覆盖了该行业最耗时的任务：制作 Pitchbook、KYC 文件筛查以及月末结账。每个模板都作为 Claude Cowork 和 Claude Code 中的**插件**以及 Claude Managed Agents 的**食谱（cookbook）**提供，目标是让团队能够在数天而非数月内将 Claude 投入实际的金融工作。

## 十大 Agent 模板详解

### 研究和客户覆盖（Research and Client Coverage）

**1. Pitch Builder（演示构建器）**
- 创建目标列表
- 运行可比公司分析
- 起草客户会议的 Pitchbook

**2. Meeting Preparer（会议准备者）**
- 在通话前汇编客户和交易对手的简报

**3. Earnings Reviewer（财报审查者）**
- 阅读财报电话会议记录和文件
- 更新模型
- 标记与投资论点相关的变化

**4. Model Builder（模型构建者）**
- 从文件、数据源和分析师输入创建和维护财务模型

**5. Market Researcher（市场研究员）**
- 跟踪行业和发行人发展
- 综合新闻、文件和券商研究报告
- 为信贷和风险审查标记项目

### 财务和运营（Finance and Operations）

**6. Valuation Reviewer（估值审查者）**
- 检查估值是否与可比公司、方法论和公司的审查标准一致

**7. General Ledger Reconciler（总账对账员）**
- 对账总账账户
- 根据账簿运行净资产价值计算

**8. Month-end Closer（月末结账员）**
- 运行结账清单
- 准备日记账分录
- 生成结账报告

**9. Statement Auditor（报表审计员）**
- 审查财务报表的一致性、完整性和审计准备度

**10. KYC Screener（KYC 筛查员）**
- 汇编实体文件
- 审查源文件
- 将升级材料打包以供合规审查

## 部署方式

### 作为插件（Plugin）

在 Claude Cowork 或 Claude Code 中，模板与分析师一起运行，使用桌面上的现有软件。将 Pitch Agent 交给目标列表，可以返回：
- Excel 中的可比模型
- PowerPoint 中起草的 Pitchbook
- Outlook 中准备好的附函

### 作为托管 Agent（Managed Agent）

同一模板在 Claude Platform 上自主运行，用于跨越整本交易簿或夜间计划的工作。食谱将其建立，包含公司否则需要自己工程的构建块：
- 可工作数小时的长运行会话
- 每个工具的权限
- 托管凭证库
- Claude Console 中的完整审计日志

**关键点：** 在两种场景中，用户都坚定地保持在循环中——审查、迭代和批准 Claude 的工作，然后才将其发送给客户、归档或执行。

## 跨 Microsoft 365 集成

Claude 现在可以通过插件直接在 Microsoft Excel、PowerPoint、Word 和 Outlook 中工作：

- **Outlook：** 可以充当幕僚长，分流你的收件箱、安排会议并按照你的声音起草回复。
- **Excel：** 从文件和数据源构建财务模型，审计链接工作簿中的公式，运行敏感性分析。
- **PowerPoint：** 起草当基础数字变化时自动更新的演示文稿。
- **Word：** 根据公司自己的模板编辑信贷备忘录。

**重要特性：** Claude 在所有四个平台之间携带其知识和上下文：在 Excel 中开始模型的分析师不需要当该工作转移到 PowerPoint 时重新解释它。

### Dispatch 功能

在 Claude Cowork 中，用户还可以通过文本或语音从任何地方分配 Claude 工作任务。Claude 可以在分析师离开办公桌时继续处理本地文件，当他们回来时完成的工作已准备好审查。

## 生态系统扩展

### 现有连接器

Claude 连接到数十个市场数据、研究平台和金融机构的内部系统，包括：
- FactSet, S&P Capital IQ, MSCI, PitchBook
- Morningstar, Chronograph, LSEG, Daloopa
- 以及公司自己的数据仓库、研究存储库和 CRM

### 新增连接器

**1. Dun & Bradstreet**
提供经过验证的企业身份全球标准，帮助企业连接记录系统并扩展 AI 启用的工作流程。

**2. Fiscal AI**
扩展跨公开股票的实时基本面覆盖，以进行更深入的研究和基准测试。

**3. Financial Modeling Prep**
提供跨股票、ETF、加密货币、外汇和大宗商品的实时报价、基本面、报表、文件和记录。

**4. Guidepoint**
搜索 100,000+ 经过合规审查的专家访谈记录，并提供链接到源逐字摘录。

**5. IBISWorld**
跟踪数千个行业的收入、财务比率、风险评分、成本结构和预测。

**6. SS&C IntraLinks**
授予 Claude 访问 DealCentre 数据室的权限，用于文档搜索、尽职调查问答和交易活动跟踪。

**7. Third Bridge**
授予 Claude 访问有关公司、行业和价值链的主要来源专家访谈的权限。

**8. Verisk**
提供用于承保、理赔和风险分析的财产、意外和特种保险数据。

### 新增 MCP 应用

**Moody's** 推出了一个 MCP 应用，将专有信用评级和超过 6 亿家公开和私人公司的数据用于合规、信贷分析和业务发展。

## 认知价值

这次发布标志着 AI Agent 从"通用工具"向"行业专用解决方案"的重要转变：

1. **深度行业化：** 十个模板精准覆盖金融服务业的核心工作流，显示了 Anthropic 对该行业的深入理解。

2. **集成优先：** 不是要求金融机构改变工作方式，而是将 Claude 深度集成到他们已有的工具链（Microsoft 365、专业数据源）中。

3. **企业级保障：** 提供托管 Agent、审计日志、权限控制等企业必需的安全和合规特性。

4. **生态系统策略：** 通过连接器（Connectors）和 MCP 应用与行业数据源深度集成，而非要求企业自己解决数据访问问题。

5. **以人为本的设计：** 强调用户"保持在循环中"，Agent 是增强而非替代专业判断。

## 对行业的影响

1. **降低 AI 采用门槛：** 企业无需从零开始构建 Agent 能力，可以快速部署并在实际工作中验证价值。

2. **重新定义"金融科技"：** 从软件解决方案转向 AI Agent 解决方案，从"工具"转向"数字员工"。

3. **加速人才转型：** 金融专业人员将从重复性任务中解放，专注于更高价值的判断和决策工作。

4. **新的竞争维度：** AI 能力将成为金融机构技术栈的核心竞争力，类似之前的数字化和移动化浪潮。

## 结论

Anthropic 的这次发布不是简单的产品更新，而是对 AI Agent 在企业级应用中如何落地的一次系统性回答。它展示了"AI Agent"从概念走向量产的路径：

- **具体而非通用：** 针对具体行业和工作流
- **集成而非替代：** 深度融入现有工具和流程
- **增强而非取代：** 放大而非替代人的能力
- **系统而非单点：** 涵盖模型、工具、数据、安全的完整解决方案

对于金融服务行业，这意味着 AI Agent 时代已从"未来时"变为"现在时"。对于其他行业，这提供了一个可参考的落地路径模板。
