# SWE-CI：AI Agent 的长期维护能力大考

> **字数**：约 1050 字
> **风格**：专业、数据驱动、适度类比

---

AI coding agent 的能力在过去两年突飞猛进，从 HumanEval 到 SWE-bench，基准测试推动了模型快速进步。但这些评测都有个根本缺陷：**快照式范式**——给个 bug 描述，agent 一次性生成修复代码，通过测试就得高分。

这就像让学生做数学题，答案对错立判。但真实的软件工程远非如此。

**软件维护占整个生命周期成本的 60-80%**。Lehman's Laws 指出，软件质量会随着维护进程而固有退化。一个能在 SWE-bench 上得高分的 agent，可能在长期维护中表现糟糕——因为它缺乏长期规划能力，容易引入技术债务，无法在多次修改中保持代码一致性。

问题的核心在于：**可维护性无法在单一时间点观察**。ISO/IEC 25010 标准明确定义，可维护性是"软件能够被有效修改而不引入缺陷或降低现有质量的程度"。这个特性只有在连续修改中才能显现。

## SWE-CI：首个基于 CI 循环的评测基准

中山大学与阿里巴巴团队提出的 **SWE-CI（SoftWare Engineering – Continuous Integration）**，填补了这一空白。

**核心创新：从 snapshot 到 evolution**

与现有基准的快照式范式不同，SWE-CI 采用演进式范式。agent 不是一次性生成代码，而是在数十轮迭代中逐步演进代码库，每一轮的决策都会影响后续轮次。

**Architect-Programmer 双 agent 协议**

SWE-CI 引入双 agent 评测协议，模拟真实软件团队的 CI 循环：
- **Architect Agent**：识别功能缺口，生成高层需求文档（每次不超过 5 个最紧急需求）
- **Programmer Agent**：根据需求修改代码

这确保了迭代式开发：Architect 不会一次性给出所有需求，而是让 Programmer 在快速、聚焦的开发中逐步推进。

**EvoScore：未来加权的评分指标**

为量化长期可维护性，SWE-CI 引入 EvoScore。核心思想是**未来加权**：后期迭代获得更高权重（通过参数 `γ ≥ 1` 控制）。

当 `γ > 1` 时，后期迭代表现更好的 agent 得分更高。这直接呼应了可维护性的定义：真正可维护的代码库是随着演进进程仍然易于修改的代码库。

**数据规模**

- 100 个评测任务，来自 68 个不同 Python 仓库
- 每个任务平均跨越 233 天演进历史，涉及 71 个连续提交
- 至少 500 行源代码修改（不含测试文件）

## 关键发现：当前 LLM 的长期维护能力仍不足

研究团队评测了 8 个提供商的 18 个模型，消耗超过 100 亿 tokens。三个关键观察：

**1. Claude Opus 领先，GLM-5 突出**

在所有评测模型中，Claude Opus 系列表现最佳，GLM-5 也表现突出。更重要的是，同一提供商家族内，新模型总是优于旧模型，且 2026 年后发布的模型提升幅度明显更大。

**2. 不同提供商对可维护性重视程度不同**

通过调整 `γ` 参数，研究发现：
- **MiniMax、DeepSeek、GPT** 偏好长期收益（`γ > 1` 时排名上升）
- **Kimi、GLM** 偏好短期回报（`γ < 1` 时排名更高）
- **Qwen、Doubao、Claude** 相对稳定

这反映了不同提供商训练策略的差异。

**3. 零回归率普遍低于 0.25 —— 关键警示**

回归是软件质量稳定性的核心指标：如果单元测试在代码修改前通过但修改后失败，就发生了回归。

SWE-CI 测量了零回归率（整个维护过程中无回归的样本比例）。结果令人警醒：**大多数模型的零回归率低于 0.25，只有 Claude Opus 系列的两个模型超过 0.5**。

这意味着，当前 LLM 在完全自动化、长期、多轮软件开发维护场景中，仍然面临巨大挑战。

## 影响与局限

**对 AI coding agent 发展的意义**

SWE-CI 可能成为新标杆，推动领域从短期表现转向长期稳定性，从工具属性转向协作伙伴。我们可能会看到专门针对长期维护优化的 agent 架构，记忆和规划模块成为标配。

**对企业采用的启示**

不是看单次修复成功率，而是看长期维护的稳定性。这能帮助企业更现实地预期 AI agent 的能力边界。

**局限**

- 数据规模：100 个任务 vs SWE-bench 的 2,294 个
- 语言限制：仅 Python
- 评估成本：多轮迭代评测耗时耗资源

## 结语

SWE-CI 的意义不仅在于提供一个新基准，更在于**重新定义了 AI coding agent 的能力标准**。在 AI agent 日渐普及的今天，我们需要的不是会写代码的工具，而是能与我们一同成长、维护代码库的伙伴。SWE-CI，正是朝这个方向迈出的重要一步。
