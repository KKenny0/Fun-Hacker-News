## 🔍 核心内容分析

### 1. 概念提出
Axel Delafosse 提出了一个革命性的想法：让 Claude 和 Codex 作为配对程序员一起工作，一个作为主要工作者，另一个作为代码审查者。这种模式模仿了人类的协作方式，让 AI 模型能够像人类程序员一样进行代码协作。

### 2. 技术实现
作者开发了一个名为 `loop` 的简单 CLI 工具，具有以下特点：
- 在 tmux 中并启动 claude 和 codex
- 通过桥梁机制让两者能够互相交流
- 保持迭代间的上下文连续性
- 支持用户参与循环，可以引导、回答问题

### 3. 实际发现
在构建代码审查代理时，作者发现：
- Claude 和 Codex 提供不同的反馈意见
- 即使反馈相同，这也是一个强烈的信号（表明两人都同意）
- 团队能够处理 100% 的反馈，当两个审查者意见一致时

### 4. 协作优势
- **更快的反馈循环**：相比传统的代码审查，速度更快
- **更自然的交互**：AI 之间的交互更加自然
- **保持上下文**：在迭代间保持上下文连续性
- **主动性增强**：AI 更加主动，能够自主进行交互

### loop 工具特点
- **简单易用**：死简单的命令行工具
