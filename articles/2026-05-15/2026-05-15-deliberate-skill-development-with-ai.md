# 当 AI 让你变「聪明」时，它也在悄悄偷走你的学习机会

## 一、一个令人不安的现实

想象一下：你正在使用 Claude Code 或 Codex 开发一个新功能。AI 帮你生成代码、解释架构、甚至帮你重构。几个小时后，功能完成了，你感觉很好 —— 代码写得漂亮，AI 回答了所有问题。

但当你第二天回到这个项目，试图独立实现类似功能时，你发现自己卡住了。你能认出 AI 生成的代码，却说不出为什么这样设计；你知道某个函数调用了另一个，却想不起背后的设计决策；你甚至不确定自己是否真正理解了代码的工作原理。

这不是你一个人的经历。这是一个普遍存在的现象，而 Hacker News 上最近一个获得 216 票的项目（[A Claude Code and Codex Skill for Deliberate Skill Development](https://github.com/DrCatHicks/learning-opportunities)）正是为了解决这个问题而生。

## 二、AI 辅助编程的学习陷阱

这个项目的创建者 Dr. Cat Hicks 指出了一个关键问题：**AI 编程工具可能带来特定的学习风险，这些风险可以从基础的学习科学原理中预测出来**。

具体来说，AI 工具可能加剧以下五个学习陷阱：

### 1. 生成效应（Generation Effect）

学习科学告诉我们，自己生成内容比被动接受理解更深。但 AI 编程工具恰恰相反：它鼓励你接受生成的代码，而不是自己动手写。这跳过了构建理解所必需的主动处理过程。

### 2. 流畅性错觉（Fluency Illusion）

干净、可读的 AI 生成代码会让你误以为自己理解了它。同样，通过搜索轻松获得的知识会营造「知识掌握」的错觉和更完整心智模型的错觉。

### 3. 间隔效应（Spacing Effect）

机器的效率可能推动你走向持续填鸭式的学习和长时间的生产会话，而缺乏长期保持记忆所需的学习节奏、反思和间隔。

### 4. 元认知（Metacognition）

快速工作流往往没有留下空间来监控学习和发展图式表征，以及用户在使用新技术时对自己相对专业水平和知识的感知。

### 5. 测试和检索（Testing and Retrieval）

代理模型倾向于给出完整答案，这可能导致用户错失从自我测试和检索新知识特定组件中获益的机会，而这些能加强记忆保持。

## 三、一个科学干预方案

Dr. Cat Hicks 开发的这个技能（skill）试图通过引入学习科学验证的技术来对抗这些风险。核心思路是：**在你在 AI 辅助下完成架构工作（新文件、模式更改、重构）后，Claude 会提供可选的 10-15 分钟学习练习**。

这些练习扎根于证据学习科学，使用预测、生成、检索练习和间隔重复等技术，为你提供来自自己项目工作的半完成例子。

### 它是如何工作的

当你完成重要工作（创建新文件或模块、数据库模式更改、架构决策或重构、实现不熟悉的模式、任何你在开发过程中提出「为什么」问题的工作）后，Claude 会问：

> 「你想做一个关于 [主题] 的快速学习练习吗？大约 10-15 分钟。」

如果你接受，Claude 会带你完成一个互动练习。一个关键设计原则：**Claude 暂停并等待你的输入**，而不是回答自己的问题。这可能会感到令人沮丧，但这推动了 Claude 默认总是提供完整答案的行为，并鼓励你自己的心理投入和学习。

### 练习类型

1. **预测 → 观察 → 反思**：你预期会发生什么？现在让我们看看。什么让你惊讶？

2. **生成 → 比较**：在看到实现之前，你将如何处理这个问题？

3. **追踪路径**：一步步执行，预测每个转换

4. **调试这个**：这里会出什么问题，为什么？

5. **回授（Teach it back）**：像给新开发者入职一样解释这个组件

6. **检索检查**：在会话开始时，你记得上次的内容吗？

### 抑制条件

为了防止过度打扰，Claude 在以下情况下不会建议学习机会：

- 你在本会话中已经拒绝了一次练习
- 你在本会话中已经完成了 2 次练习

## 四、背后的科学支撑

这些练习借鉴了学习科学的成熟发现，以及关于典型学习者误解的大量研究。设计选择也源自与多位开发者的定性访谈，了解他们在快速代理编码中觉得最令人沮丧、担忧或困难的方面。

项目的 PRINCIPLES.md 文件详细解释了这些原理，包括：

- **Bjork, Dunlosky, & Kornell (2013)** 关于自我调节学习的研究
- **Dunlosky et al. (2013)** 关于高效学习技术的心理学研究
- **Ericsson, Hoffman, & Kozbelt (2018)** 关于专家表现的研究
- 以及许多其他关于认知负荷、测试效应、间隔重复的经典研究

Dr. Cat Hicks 本人在开发者繁荣和 AI 技能威胁方面也进行了大量实证研究。她的研究涉及数千名开发者，发现对学习的强烈价值和承诺预测着开发者在想象需要适应代理编码时感到更少的威胁、担忧和焦虑。

## 五、为什么这很重要

这个项目不仅仅是一个技术工具，它代表了我们在 AI 时代重新思考学习的一种尝试。

当 AI 工具变得越来越强大时，一个关键问题是：我们是在用 AI 来增强我们的能力，还是在用 AI 来替代我们的思考？这个项目给出了一个明确的答案：**AI 应该是增强我们学习过程的工具，而不是替代我们学习的捷径**。

更重要的是，它提醒我们：学习不是被动接受知识的过程，而是一个主动建构理解的过程。AI 可以为我们提供信息和指导，但真正的理解和掌握仍然需要我们自己的心理投入和反思。

## 六、给开发者的建议

如果你想尝试这个技能，可以从以下步骤开始：

### 安装（Claude Code）

1. 添加市场：
   ```
   /plugin marketplace add https://github.com/DrCatHicks/learning-opportunities.git
   ```

2. 安装插件：
   ```
   /plugin install learning-opportunities@learning-opportunities
   ```

3. 重启 Claude Code 以激活

### 可选功能

- **自动提示**：在每次 git 提交后自动考虑提供练习
- **仓库方向（Orient）**：创建包含建议课程的 orientation.md 文件，帮助你学习新仓库

### 团队使用

如果你想在团队中尝试这个技能，项目还提供了一个配套的轻量级前后测量手册（[MEASURE-THIS.md](https://github.com/DrCatHicks/learning-opportunities/blob/main/learning-opportunities/docs/MEASURE-THIS.md)），包括：

- 来自开发者繁荣和 AI 技能威胁同行评审研究的验证调查项
- 关于如何（以及如何不）使用结果的指导
- 「团队吹嘘」模板，将你的实验打包成发给领导的邮件

## 七、结语

在 AI 时代，学习能力比以往任何时候都更重要。当我们越来越依赖 AI 工具时，确保我们真正学习和理解变得更加关键。

这个项目提醒我们：AI 不是学习的替代品，而应该是学习的增强品。通过有意识地融入学习科学验证的技术，我们可以在享受 AI 工具带来的效率提升的同时，避免掉入学习陷阱，真正提升我们的专业能力。

正如 Dr. Cat Hicks 所说：「建立你的专业知识，而不仅仅是你的项目。」
